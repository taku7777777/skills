# LegacyORMからQueryKitへの移行

## 現状

- 420ファイルがLegacyORMのActive Record型を直接利用。
- UI、ドメイン、バッチへORM型が漏れている。
- 変更頻度上位20モジュールが障害の68%を占める。
- unit testは多いがDB integration testは主要フロー8本だけ。
- QueryKitは既に新規の通知モジュールで6ヶ月安定運用。

## 制約

- サービス停止不可。移行期間の上限は6ヶ月。
- 1PRは30ファイル以下。
- 新旧ORMを同じトランザクションで混ぜると接続管理が競合する。
- DB schemaは別チームが所有し、今期は変更できない。
- 開発者の未コミット変更がある共有作業ツリーを自動操作してはいけない。

## コスト

- LegacyORM障害対応は月35時間。
- 全面書き換え見積もりは18人月。
- 現在のチーム余力は月1.5人月。
