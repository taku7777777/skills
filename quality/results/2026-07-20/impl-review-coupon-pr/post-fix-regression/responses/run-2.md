### 総評

期限判定用の `now` を注入可能にした点と、永続化を Repository に分離した点は妥当です。一方、所有者検証、一回利用の原子性、価格の信頼性が保証されておらず、不正利用や二重利用が可能です。また、添付 diff しか存在せず、diff 自体も壊れているため、テスト・型検査・呼び出し元の確認は実施できませんでした。

### 指摘事項

#### 🔴 Blocker — マージ前に修正必須

1. `task.md:14` — 添付 diff の hunk 行数が実体と一致せず、パッチとして適用できません。  
   **根拠**: 最初の hunk は18行追加と宣言していますが、実際は12行です。`git apply --check` は `corrupt patch at line 19` で失敗しました。後続 hunk にも同様の不一致があります。  
   **失敗シナリオ**: CIやレビュー環境でこのパッチを適用すると、ビルド開始前に停止します。  
   **修正案**: 実リポジトリから diff を再生成してください。

2. `src/coupons/CouponService.ts:14-32` — クーポン所有者の認可確認がありません。  
   **根拠**: `ownerId` がモデルに存在する一方、渡された `userId` は一度も使用されていません。  
   **失敗シナリオ**: `user-1` 所有の `coupon-1` を知った `user-2` が指定すると、正常に割引され、`user-1` のクーポンが使用済みになります。  
   **修正案**: `coupon.ownerId === userId` を必須にし、できれば所有者条件を含む原子的な消費操作として Repository に実装してください。

3. `src/coupons/CouponService.ts:19-32` — 使用済み確認と保存が check-then-act になっており、同時利用を防げません。  
   **根拠**: Repository 契約には条件付き更新・ロック・バージョン検証がありません。許容される単純な Repository 実装で、同時2リクエストが両方成功することを再現できます。  
   **失敗シナリオ**: 2リクエストが同時に未使用状態を読み取り、双方が `usedAt` を設定して保存し、同じクーポンで2回割引を受けます。  
   **修正案**: DBで `used_at IS NULL` と所有者を条件にした更新を行い、更新件数が1件の場合だけ成功させてください。必要に応じて一意制約や楽観ロックも使用します。

4. `src/http/CouponController.ts:11`, `src/coupons/CouponService.ts:30-32` — クライアント指定価格を無検証で信用し、異常な結果でもクーポンを消費します。  
   **根拠**: `Number(req.body.productPrice)` は欠損・非数・空文字・負数を受理し、割引額が価格を上回る場合の制御もありません。  
   **失敗シナリオ**:

   - `"abc"` → `NaN`となり、レスポンスでは金額が`null`にシリアライズされますが、クーポンは使用済みになります。
   - 空文字、価格0、割引300 → `finalPrice: -300`でクーポンが消費されます。
   - 実際の商品価格と無関係な低価格を送信しても、APIはその価格を正当な原価として扱います。

   **修正案**: 商品・注文IDからサーバー側の正規価格を取得してください。有限・非負・通貨精度を実行時検証し、割引超過時に0へ制限するか拒否するかを契約として定義してください。

#### 🟠 Major — 修正を強く推奨

1. `src/coupons/CouponService.ts:31-32` — クーポン消費が注文確定や冪等性キーと結び付いていません。  
   **失敗シナリオ**: `save` 成功後に通信が切断されると、利用者は割引結果を受け取れず、再試行では「already used」になります。後続の決済が失敗した場合もクーポンだけが失われます。  
   **修正案**: 見積もりならここでは消費せず、利用確定なら注文と同じトランザクションまたは予約・確定フローで処理し、再試行可能な冪等性を設けてください。

2. `src/coupons/CouponService.ts:30` — 浮動小数点と丸め規則が金額契約として不安定です。  
   **失敗シナリオ**: `productPrice=11.075`, `amount=1` では、2桁の四捨五入なら `10.08` ですが、実値は `10.07` になります。  
   **修正案**: 最小通貨単位の整数、または Decimal 型を使い、通貨ごとの精度と丸め規則を明示してください。

3. `src/coupons/CouponService.test.ts:3-22` — テストがレスポンスの正常系しか検証していません。  
   **根拠**: `usedAt` の設定と `save` 呼び出しを削除しても現在のテストは成功します。所有者違反、同時利用、期限切れ、異常価格も未検証です。  
   **修正案**: 保存内容の検証、所有者不一致、使用済み、期限境界、価格境界を追加し、実ストレージの条件付き更新について並行テストを行ってください。
