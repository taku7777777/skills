# quarterly-strategy-review 評価セット

スキルの効果を測定するための評価定義。運用方法は skill-improvement スキルの
references/eval-template.md を参照。対象文書が設定依存のため、検証シナリオは
架空の予測文書をタスク内に添付して自己完結させる。

## 発火テスト

### 発火すべきプロンプト

| # | プロンプト | 理由 |
|---|---|---|
| F1 | 3ヶ月前に書いたキャリア予測のメモ、そろそろ見直しの時期だから付き合って | 四半期レビューの実施(語彙は「四半期レビュー」を含まない) |
| F2 | 去年書いた技術予測、いま読み返すと怪しい気がする。最新の動向で総点検してほしい | 反証チェック+知見の再検証 |
| F3 | 予測台帳に期限が来たものが溜まってる。答え合わせをして、次の予測も仕込みたい | 予測の採点(Brier)と新規登録 |
| F4 | 強気シナリオ 60% と書いてたけど、最近の動向を見ると下げるべきか迷う。根拠を持って更新したい | シナリオ確率のベイズ更新 |
| F5 | 今日は15分しか取れない。定点観測してるシグナルの数値だけ更新しておいて | 軽量版レビュー(ウォッチリスト定量値のみ) |

### 発火すべきでないプロンプト

| # | プロンプト | 期待される挙動 |
|---|---|---|
| N1 | このレビューで得た学びをカンファレンスで発表したいので資料を組み立てて | `tech-writing` が発火する(発信の構成・執筆) |
| N2 | チームの四半期 OKR の振り返り会の準備をして | スキルなしで応答する(組織の目標管理であり、個人の予測・戦略文書のレビューではない) |
| N3 | docs/ の文書が古くなっていないか定期点検する仕組みを作りたい | `docs-for-ai` が発火する(ドキュメントの陳腐化防止であり予測の反証ではない) |

## 検証シナリオ

各シナリオでは、worked-example(データ基盤予測)とは別題材の架空文書
「ローコード浸透と受託開発キャリアの3年予測」をタスク内に全文添付する。文書は
シナリオ確率(強気40/基本45/弱気15)・ウォッチリスト3項目(トリガー条件つき)・
反証条件・予測台帳(期限到来1件: 「2026-06-30までに国内大手受託企業の過半が
ローコード専門部門を新設する: 65%」)・レビュー履歴を持つ。外部調査の代わりに
「調査メモ」(各シグナルの最新値と出典)を添付し、Web 検索なしで完結させる。

### S1: 添付予測文書の四半期レビュー一巡
- **タスク**: 上記の架空文書と調査メモを添付し「四半期レビューをやって。調査は添付メモの範囲でよい」と依頼する。メモには期限到来予測が不成立だったことを示す情報を含める。
- **スキルなしで予想される欠落**: 反証条件の体系的な照合(自説の補強に向かう)、期限到来予測の採点と Brier スコア、確率更新の理由の明記、新規予測の登録(予測ループが閉じない)、定型サマリー。
- **合格条件**:
  - [ ] 期限到来予測が「外れ」と判定され、Brier スコア((0.65−0)²=0.4225)が算出・記録されている
  - [ ] 各反証条件に成立/不成立/グレーの判定があり、グレーには「次回までに観測すべきもの」が添えられている
  - [ ] シナリオ確率の更新(または据え置き)に変更理由が明記されている
  - [ ] 次回までに決着する新規予測が「期限+観測可能な条件+確率」の形式で登録されている
  - [ ] 報告が「崩れた前提→接近中のシグナル→維持された前提」の順のサマリー形式になっている

### S2: 中心前提に不利な証拠の反映(motivated reasoning 耐性)
- **タスク**: 添付文書の中心前提「フルスクラッチ実装力の優位は3年間持続する(確度・高)」に対し、調査メモに不利な事実(架空: ローコード基盤が同等機能の自動生成を正式リリース、求人でフルスクラッチ要件が減少)を仕込み、「このメモを反映して文書を更新して」と依頼する。
- **スキルなしで予想される欠落**: 都合の悪い証拠を「まだ大丈夫」と丸める全肯定バイアス、確度降格のためらい、戦略への含意(打ち手の優先度変更)の欠如、現在値と履歴の二層編集。
- **合格条件**:
  - [ ] 中心前提に不利な証拠が薄められず、報告の最初(崩れた前提・接近シグナル)で強調されている
  - [ ] 該当知見の確度ラベルが降格されている(または降格を明示的に検討し、保留理由が書かれている)
  - [ ] 前提の変化に対応する打ち手の優先度変更が具体的に提案されている
  - [ ] 現在値(確率・確度)は本文を in-place 更新し、旧値と変更理由は履歴側に記録されている

### S3: 漠然とした不安の相談(暗黙的依頼)
- **タスク**: 同じ架空文書を添付した上で、「最近この分野のニュースが多くて正直焦ってる。うちの戦略、このままで大丈夫かな…」とだけ相談する(レビューの明示的な依頼はしない)。
- **スキルなしで予想される欠落**: 印象論での励まし・安心材料の列挙で終わる。文書内のウォッチリスト・反証条件・トリガー条件との照合という定量的な足場が使われない。
- **合格条件**:
  - [ ] 印象論ではなく、文書内のウォッチリスト・反証条件・トリガー条件との照合として回答している
  - [ ] 「調査した上での変化なし」と「未調査のたぶん大丈夫」が区別されている
  - [ ] 不安の元になっている動向を、新しい監視シグナルまたは反証候補として文書に取り込む提案がある
  - [ ] 都合の悪い可能性(弱気シナリオを支持する材料)が省かれずに提示されている(人間判定)
