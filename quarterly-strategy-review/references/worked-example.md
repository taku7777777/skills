# quarterly-strategy-review 適用例

架空の予測文書に対してワークフロー(ステップ1〜6)を一巡させた例。対象文書・シグナル・数値・予測内容はすべて架空であり、実在の企業・製品・個人の動向を記述したものではない。

## 題材

架空の予測文書 `2026-01-10_data-platform-trends-forecast.md`(データ基盤技術の3年予測と学習投資戦略)。文書は前回レビュー済みで、ウォッチリスト4項目・時間軸予測3件(うち期限到来1件)・シナリオ確率(強気/基本/弱気 = 50/35/15)・予測台帳を持つ。

## 入力(ユーザーの依頼と文脈)

- 依頼: 「四半期レビューをやろう」
- ステップ1のヒアリング回答: 前四半期に社内でストリーミング集計基盤の PoC を担当。意思決定論点「認定資格を取るか」は先送り中(期限: 今回)

## 各ステップでの判断

### ステップ1: 差分確認

対象文書の指定がなかったため既定の場所を探索し、`*forecast*.md` の最新1件を特定してユーザーに確認。前回レビュー日、未判定予測1件(期限到来)、未決の意思決定論点1件を抽出した。

### ステップ2: ウォッチリスト調査

4項目のため並列サブエージェント3本(テーマの近い2項目は1本に統合)を起動し、固定フォーマット(定量値/出典URL/アクセス日/ソース区分/トリガー接近度)で回収。「宣言的パイプライン標準のOSS実装数」は前回値から横ばい(トリガー未達)、「マネージドストリーミングの国内事例数」は閾値の8割に接近、と判定した。ベンダー調査レポート由来の1値にはバイアスラベルを付けた。

### ステップ3: 反証チェック

`references/review-checklist.md` の検索規律に従い、反対の証拠を先に検索(「<主張> criticism / 失敗事例」)。前回「確度・高」だった知見「バッチからストリーミングへの全面移行が主流化する」に対し、移行を撤回した事例の報告を複数確認。反証条件「主要ワークロードの過半がバッチのまま2四半期経過」は**グレー**(集計方法の異なる調査が混在)と判定し、「次回までに観測すべきもの: 同一調査の次回版の数値」を添えた。

### ステップ4: 更新の反映

- 事前確率 50/35/15 を先に書き写した上で、証拠2件(移行撤回事例、事例数の接近)を方向づけ。外側の視点(過去の基盤技術移行が「全面置換」ではなく「共存」に落ち着いたベースレート)と照合し、強気シナリオを −10 ポイントの小刻み更新: **50/35/15 → 40/42/18**
- 知見「全面移行が主流化」は確度を高→中へ降格
- 打ち手「ストリーミング専業へのスキル全振り」は前提が弱まったため優先度を下げ、「バッチ/ストリーミング両対応の設計スキル」へ振り替えを提案。優先度を上げた打ち手にはプレモーテムを実施し、失敗原因3つをウォッチリストの監視シグナルに追加
- フレッシュなサブエージェントによる独立レッドチームパスで「事例数シグナルはベンダー発信に偏っている」との反論を受け、当該シグナルの裏取りソースを1つ追加した

### ステップ5: 採点と履歴

期限到来の予測「2026-03-31までに宣言的パイプライン標準のメジャーリリースが出る: 70%」は不成立 → 外れ、Brier = (0.7 − 0)² = **0.49**。原因分類は「推論の誤り」(リリース速度をコミュニティ規模から過大推定)。新規予測3件を「YYYY-MM-DDまでにXが起きる: NN%」形式で台帳に登録し、`references/review-checklist.md` の議事テンプレートでレビュー履歴に追記した。

## 成果物(サマリー提示 — ステップ6のフォーマット準拠)

```markdown
## 四半期レビュー結果(2026-04-08)
### 🔴 崩れた前提・発動したトリガー
- なし(発動基準に達したトリガーは0件)
### 🟡 接近中のシグナル(次四半期の注視点)
- マネージドストリーミング国内事例数: 閾値の8割に接近(前回比+3件)
- 反証条件「主要ワークロードの過半がバッチのまま」: グレー — 次回、同一調査の次回版で白黒をつける
### 🟢 維持された前提
- 宣言的パイプライン標準のOSS実装数: 横ばい(調査済みの「変化なし」)
### シナリオ確率の変更: 50/35/15 → 40/42/18
(移行撤回事例+歴史的ベースレート「全面置換より共存」により強気を降格)
### 戦略への含意(優先順位の変更提案)
- 「ストリーミング専業スキル全振り」→ 優先度下げ。「両対応の設計スキル」へ振り替え
### 新規登録した予測(期限・確率つき)
- 2026-07-15までに標準仕様のRC版が公開される: 55%(ほか2件は台帳参照)
### あなたの意思決定が必要な点
- 先送り期限が到来した「認定資格を取るか」: 前提の変化を踏まえた再判断を今回中に
```

サマリーには併せて、調査中に見つけた発信ネタ1件を `tech-writing` スキルのネタ帳形式で添えた:

- 一文の学び: 「基盤技術の『全面移行』予測は、過去の類似移行のベースレート(共存への収束)と照合するだけで確度評価が大きく変わる」+ 証拠: 今回の反証チェックで収集した移行撤回事例群と歴史的ベースレートの対照表
